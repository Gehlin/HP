# Phase 02: Convert `session.ts` and Its 13 Callers

**This is the highest-risk phase in this playbook.** `src/utils/session.ts` is the core session/exam-building logic — it's called from 13 different files, including the actual practice/exam-taking critical path (`Session.tsx` itself, `Practice.tsx`, `Resultat.tsx`). Read `Phase-01-Shared-Loader-and-Simple-Consumers.md`'s completion notes first (the shared `loadQuestions()`/`useQuestions()` utilities already exist — use them, don't rebuild).

A broken conversion here doesn't crash — it shows as blank content, a stuck loading state, or a session that silently fails to build. **Verify every touched page live with Playwright, not just `npm run build` passing.**
