# Phase 01: Design Exploration — Top Bar & Home Declutter

**This phase does not touch the live app's routing, existing `TopNav.tsx`, or any real page yet.** Produce concrete, viewable design directions first; a later phase wires the chosen one in everywhere. Read the playbook `README.md` first for full context and the user's stated problem.

The app's existing visual language (established across the earlier `2026-06-27-Hgskoleprovet-Design-System` playbook and its design tokens in `src/index.css`) must carry through: paper background `#F1ECE3`, forest green `#224A3A`, terracotta `#BF5A33`, gold `#E4C66A`, Newsreader serif for headings/numbers, Hanken Grotesk sans for everything else. "Impressive" means executed with real polish within this existing language — not a different visual style bolted on top.

## Constraints to design within

- Mobile-width app (~390–402px usable viewport, confirmed via all prior design/verification work in this repo).
- Logo top-left, must link to `/` (Hem) when clicked.
- Primary nav (Hem/Öva/Statistik/Profil) needs to move from the bottom into this top bar, one-tap accessible (no hamburger/drawer for primary nav — that adds friction for the 4 most-used destinations in the app).
- Must still hide/adapt appropriately on the same paths the current bottom bar hides on (`/session`, `/resultat`, `/granska`, `/exam/*` — check if a top bar should do something different there, e.g. show just the logo with no nav tabs, rather than fully disappearing, since a top-anchored element disappearing can cause layout jump in a way a bottom one doesn't).
