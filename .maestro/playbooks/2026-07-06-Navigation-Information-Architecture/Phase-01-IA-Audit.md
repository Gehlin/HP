# Phase 01: Information Architecture Audit

Map the app's actual current navigation structure with real data — this is the foundation everything else in this playbook depends on. Read the playbook `README.md` first (the operating principles this whole initiative works under). **This phase is investigation and reporting only — no code changes.**

The app has ~26 routes (see `src/App.tsx`'s `<Routes>` block). The new top nav bar (`AppHeader.tsx`) only surfaces 4: Hem, Öva, Statistik, Profil. Everything else — Theory/guides per subtest, the exam simulator, word builder, score predictor, SRS queue, bookmarks, settings — is reached some other way, and the working hypothesis (unverified, confirm or refute with real data) is that most of it is buried inside Profil's tool list, which is itself exactly the "nested navigation" pattern the operating principles warn about.
