**Write one table/chart set at a time, saving to the file before starting the next** — this is the pattern that fixed the stalling problem in Phases 10/11 (their earlier combined attempt stalled 3 times doing a large upfront read/plan pass; splitting and writing incrementally fixed it completely). **Mobile-width constraint (critical — this caused a real live bug fixed in Phase 07):** every new table must render without horizontal clipping at the app's ~390-402px mobile viewport. Phase 07 found column count alone doesn't predict this — cell content width matters (a 6-column table with short headers can fit; a 4-column table with a long label can overflow). Check `src/components/ChartView.tsx`'s rendering and Phase 07's completion notes for the specific column-width lessons learned, and verify every new table live with Playwright before considering it done — don't just count columns and assume it's fine.
