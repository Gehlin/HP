# Phase 02: Extract Calibration Patterns Per Subtest

Read a representative sample of the real exams gathered in Phase 01 and produce **original, derived** pattern docs for each of the 8 subtests — format templates, difficulty/topic norms, option-structure conventions — that later phases will use to audit and expand the app's question bank. Read this playbook's root `README.md` first: nothing in the output of this phase may contain verbatim real-exam text. You are extracting *statistics and structural patterns*, not quoting.

## Sampling strategy (don't read all 231 PDFs)

The post-2011 Högskoleprovet format (the 8 subtests this app models: ORD/LÄS/MEK/ELF, XYZ/KVA/NOG/DTK) has been structurally stable across 2013–2026. Reading all 27 exam sittings in full is unnecessary and unaffordable context-wise. Instead:

- Pick **6 sample dates spread across the range**: roughly one from each of 2013–2015, 2017–2019, 2021, 2023, 2025–2026 (e.g. 2013-10-26, 2017-04-01, 2019-04-06, 2021-10-24, 2023-10-22, 2026-04-18 — adjust based on what's actually in `Reference/raw-exams/MANIFEST.md` if a date is thin).
- For each sample date, use `pdftotext` (or equivalent) to extract the prov and facit PDFs to plain text once — reuse that extracted text across all 8 subtest analyses rather than re-parsing per subtest.
- If something about a subtest looks like it changed over time (option count, question style, a subtest that didn't exist in earlier years), pull 1–2 *additional* samples from that specific era to confirm — don't extrapolate from a single data point when the corpus is sitting right there.
