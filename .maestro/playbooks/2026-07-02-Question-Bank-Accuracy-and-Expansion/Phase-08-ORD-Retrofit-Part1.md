# Phase 08: ORD 5th-Option Retrofit — Part 1 (ord-e01–e98)

**User-confirmed approach:** retrofit all 168 ORD items in dedicated phases (this is part 1 of 2) rather than spreading the work across other Phase 07+ content, to avoid the drift problem Phase 06 found in MEK (where new content kept getting added in the wrong format because nobody was actively enforcing the target convention).

Every ORD question in the app currently has 4 options (A–D); every real exam sampled 2013–2026 uses 5 (A–E), per `Reference/calibration/ORD.md`. This phase adds a genuine 5th option to the 98 `ord-e01`–`ord-e98` items. This is **content-writing, not a mechanical edit** — each new option is a real word that must plausibly (but wrongly) fit "betyder ungefär detsamma som [target word]," following `ORD.md`'s distractor-construction taxonomy (near-miss register, antonym trap, same-domain-wrong-specificity, morphological false friend). Read that calibration doc in full before starting; it's the only reference for what makes a good vs. lazy 5th option here — don't invent a different taxonomy.

Read the root `README.md` for the general copyright boundary (the target words and existing 4 options are already the app's own original content, not real-exam material — you're extending original content, not deriving from real exams here — but the *taxonomy* you're following to write good distractors still comes from studying real exams, so stay disciplined about writing genuinely new words, not real exam words you might recall).
