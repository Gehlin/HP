The real findings from this phase are two documented **behavioral surprises**, both now locked in by tests so a future regression would be caught: (1) `buildSession`'s internal `groupLinkedPassages` silently deduplicates repeated ids in its input array — a caller accidentally passing a duplicate id gets a shorter list back with no warning; (2) `typeAccuracyTrend` can produce different-length, non-index-aligned arrays per question type (a session with only 1 answered question of a type contributes nothing to that type's trend, not a placeholder), which is safe today only because `Progress.tsx` happens to render each type's trend independently rather than assuming parallel arrays.
