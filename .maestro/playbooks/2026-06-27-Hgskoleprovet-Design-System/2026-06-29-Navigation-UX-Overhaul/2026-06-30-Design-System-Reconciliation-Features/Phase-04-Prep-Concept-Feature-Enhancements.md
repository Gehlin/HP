# Phase 04: Prep-Concept Feature Enhancements

With a single coherent design system in place across the app, this phase adds new functionality that deepens the Högskoleprovet prep concept for users. The codebase already has solid building blocks — `src/utils/analytics.ts` (`weakTypeSummary`), `src/utils/examDate.ts` (countdown + urgency), `src/utils/gamification.ts` (XP/streak), `src/utils/srs.ts` (spaced repetition), `src/utils/session.ts` (`buildSession`/history), and a basic weak-area drill entry point already inside `Practice.tsx` — so every task below should read and reuse that existing logic rather than re-implementing it from scratch. The goal is to extend real prep value (smarter practice targeting, exam-pacing awareness, visible consistency) using the now-consistent visual language from Phases 01–03, all driven by real stored data, not placeholder content.
