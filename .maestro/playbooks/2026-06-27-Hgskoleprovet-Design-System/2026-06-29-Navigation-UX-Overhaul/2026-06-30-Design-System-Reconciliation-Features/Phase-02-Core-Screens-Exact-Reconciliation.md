# Phase 02: Core Screens — Exact Reconciliation

With the foundation tokens corrected, this phase reconciles the five core screens from the design handoff — Hem (`Home.tsx`), Öva (`Session.tsx`), Resultat (`Results.tsx`), Statistik (`Progress.tsx`), and Profil (`Profil.tsx`/`Settings.tsx`) — against the exact component specs in `.maestro/playbooks/2026-06-27-Design-System-Implementation/DESIGN-SYSTEM-03.md` through `DESIGN-SYSTEM-07.md`. These reference docs were written purely from the design mockup and contain placeholder/hardcoded data (fake names, fake percentages, fake dates) — **do not port that placeholder data into the app**. The app already has real, working implementations of all five screens with live state, localStorage persistence, and an actual question bank — this phase fixes visual fidelity (spacing, sizing, colors, component states) only, never functionality or copy, unless a piece of real data is simply missing a visual treatment the spec describes.
